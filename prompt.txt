### Step 2 — Create unified auth context

Create `artifacts/adspot-app/src/contexts/AuthContext.tsx`:
- Uses React Query `useGetMe` pattern (from `@workspace/api-client-react`)
- Single token key: `adspot_token`
- `login(token: string)` — stores token, sets state
- `logout()` — removes token, calls `window.location.href = "/"`
- `hasRole(...roles: AppRole[])` — checks if user has any of the given roles
- `AppRole` = `"reviewer" | "brand" | "admin" | "super_admin"`
- Calls `setAuthTokenGetter(() => localStorage.getItem("adspot_token"))` at module level
- Auto-logout on 401 error via useEffect watching the query error

Create `artifacts/adspot-app/src/App.tsx` with route structure:

```
/                          → Landing (public), or redirect by role
/login                     → Login
/register                  → Register
/dashboard                 → Reviewer dashboard (requires reviewer role)

/brand/settings            → Brand settings (brand role)
/brand/analytics           → Brand portal (brand/admin roles)
/admin/dashboard           → Admin overview (admin/super_admin roles)
/admin/ads                 → Admin ads (admin/super_admin roles)
/admin/users               → Admin users (admin/super_admin roles)
/admin/events              → Admin events (admin/super_admin roles)
/admin/financials          → Admin financials (admin/super_admin roles)
/admin/panel               → Full admin panel (admin/super_admin roles)
```

Root route `/` checks `user.role` and redirects:
- admin/super_admin → `/admin/dashboard`
- brand → `/brand/dashboard`
- reviewer → `/dashboard`
- no user → shows Landing

ProtectedRoute component: if `!user` redirect to `/login`, if `roles` specified and user role not in roles, redirect to `/`
