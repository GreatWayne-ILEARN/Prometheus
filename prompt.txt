## Step 2: Create unified auth context

Create `artifacts/adspot-app/src/contexts/AuthContext.tsx` with the following behavior:

- Use the React Query `useGetMe` pattern from `@workspace/api-client-react`
- Define `AppRole` as `"reviewer" | "brand" | "admin" | "super_admin"`
- Store auth state in `localStorage` under key `"adspot_token"`
- At module level, call `setAuthTokenGetter(() => localStorage.getItem("adspot_token"))` to wire token retrieval
- Implement `login(token: string)` — stores the token in localStorage and updates state
- Implement `logout()` — removes the token from localStorage and redirects via `window.location.href = "/"`
- Implement `hasRole(...roles: AppRole[])` — returns true if the current user has any of the specified roles
- Attach a useEffect that watches the query error state and auto-logs out the user on 401 errors

## Step 3: Create unified App.tsx with route structure

Create `artifacts/adspot-app/src/App.tsx` using Wouter for routing with the following route map:

```
/                          → Landing (public), or redirect by role
/login                     → Login page
/register                  → Register page
/dashboard                 → Reviewer dashboard (requires reviewer role)
/leaderboard               → Leaderboard (requires reviewer role)
/review/:id                → Review session (requires reviewer role)
/brand/dashboard           → Brand analytics (brand/admin roles)
/brand/ads                 → My Ads list (brand/admin roles)
/brand/ads/new             → Create Ad (brand/admin roles)
/brand/ads/:id             → Ad Detail (brand/admin roles)
/brand/settings            → Brand settings (brand role)
/brand/analytics           → Brand portal (brand/admin roles)
/admin/dashboard           → Admin overview (admin/super_admin roles)
/admin/ads                 → Admin ads (admin/super_admin roles)
/admin/users               → Admin users (admin/super_admin roles)
/admin/events              → Admin events (admin/super_admin roles)
/admin/financials          → Admin financials (admin/super_admin roles)
/admin/panel               → Full admin panel (admin/super_admin roles)
```

**Root route `/` behavior:**
- If no user is logged in, show Landing
- If user exists, check `user.role` and redirect:
  - `admin` or `super_admin` → `/admin/dashboard`
  - `brand` → `/brand/dashboard`
  - `reviewer` → `/dashboard`

**ProtectedRoute component:**
- If `!user`, redirect to `/login`
- If `roles` are specified and user's role is not in that list, redirect to `/`
- Otherwise render the route
